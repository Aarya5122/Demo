Demo html file for 30days web development git class

Step 1 - Clone the repository
git clone "url"

Step 2 - Change Directory to Demo (The repository name) cd Demo(for windows)

Step 3 - Add ##Header2 to .md file which was available when we cloned the repository

Files displayed in EXPLORER column will get this alphabet towards rightmost side M --> The file which was available when cloned is modified

Step 4 - Create a new html file and write code in it.

File with .html extension displayed in EXPLORER column will get this alphabet towards rightmost side U --> That means this file is Untracked i.e it was not available in the repository we cloned.

Step 4 - Enter this command in cmd git status this tells the status of Not staged files and Untracked files

Step 6 - To stage (Putting files into staging area) the files and add Untracked files for staging and making it tracked we use git add . (. means all the files not staged or else we can use filename)

Now we can see the new added file index.html in EXPLORER on the Right-Hand side of it is changed to A from U
The warning or alert A --> The file is been added to commit

NOTE: For the first time we have to log into git using
git config --global user.email "mialid@xyz.abc"
git config --global user.name "Name"

Step 7 - We are ready to commit git commit -m "message" -m "description"
